# Node - Task & Goal Management Platform
Project Overview & Feature List

## Overview
Node is a modern, responsive, and robust task and goal management platform designed to help users organize their personal life, work projects, and academic coursework. With an emphasis on clean UI, optimistic data synchronization, and flexible data structures, Node allows users to manage multiple workspaces, track complex goals, and seamlessly integrate external tools like Google Classroom.

---

## Core Features & Functionality

### 1. Account & Authentication
- **Universal Identity:** Users can sign up securely using Email/Password or standard OAuth providers via Supabase Authentication.
- **Persistent Sessions:** Smooth session management ensuring users stay logged in securely.

### 2. Workspaces & Domains
- **Default Workspaces:** New users start with "Personal", "Work", and "Projects" workspaces to easily compartmentalize their lives.
- **Customization:** Users can create, rename, and delete custom workspaces.
- **Domain Filtering:** The dashboard allows users to filter their active tasks specifically by selected domains/workspaces.

### 3. Task Lists & Organization
- **Custom Lists:** Users can create custom task lists (projects/folders) within any workspace, assigning distinct colors for easy visual identification.
- **Smart Views:**
  - **Inbox:** Default landing place for unassigned tasks.
  - **Today:** Tasks specifically due on the current day.
  - **Upcoming:** Tasks due within the upcoming 7 days.
  - **Completed:** A retroactive view of all checked-off tasks.

### 4. Advanced Task Management
- **Task Creation:** Quick-add tasks with titles, descriptions, and dynamic properties.
- **Priorities & Status:** Assign priorities (Low, Medium, High, Urgent) and track progress via statuses (To Do, In Progress, Review, Done).
- **Due Dates & Deferrals:** Set specific due dates; the system automatically tracks how many times a task has been postponed/deferred.
- **Metadata:** Support for subtasks, custom labels, estimated completion time (in minutes), and energy level tagging.

### 5. Goal Tracking
- **Goal Creation:** Define high-level goals with target dates and descriptions.
- **Task Linking:** Link individual tasks directly to specific overarching goals.
- **Progress Tracking:** The platform automatically calculates and visualizes goal completion percentage based on the status of linked tasks.

### 6. Interactive Views
- **List View:** A clean, traditional vertical list for rapidly checking off tasks.
- **Kanban Board View:** A visual pipeline allowing users to instantly drag-and-drop tasks across custom columns (To Do -> In Progress -> Review -> Done).

### 7. Google Classroom Integration (Plugin)
- **Opt-in Connection:** Users can optionally link their Google Classroom account separately from their core Node identity.
- **Smart Import:** Automatically fetches active enrolled courses and creates corresponding distinct-colored task lists inside an "Academics" workspace.
- **Assignment Syncing:** Imports course assignments as Node tasks, cleanly handling due dates and descriptions.
- **Submission Tracking:** Node checks Google Classroom's submission states (Turned In / Returned) and automatically marks Node tasks as complete if turned in.
- **Continuous Sync:** A "Sync Now" button that silently updates assignment states without requiring repetitive account login prompts (via login_hint memory).
- **Safe Unlinking:** Users can safely disconnect Google Classroom at any time without automatically losing their imported tasks or Academic domains.

### 8. Real-Time Data & Performance
- **Optimistic UI:** Actions (completing tasks, moving kanban cards) reflect instantly on the UI while a background queue securely syncs data to the Supabase database.
- **Data Integrity:** Cascade deletion logic ensures that when a workspace or list is deleted, all underlying nested tasks are safely removed to prevent "orphaned" data.

### 9. Maintenance & Automation
- **Auto-Archival:** System logic prevents performance bloat by automatically managing and hiding completed tasks older than 30 days from the initial data hydration load.
